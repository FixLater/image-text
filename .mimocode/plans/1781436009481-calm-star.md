# Dashboard Card Redesign Plan

## Goal
Make dashboard cards display real-time status and log preview from their corresponding modules (WebSocket).

## Architecture

### Current State
- `DashboardPage`: QPushButton-based cards with icon + title + description (fixed 260x160)
- `WebSocketPage`: owns log (`QTextBrowser`), status (`statusLabel`), per-tab `logHtml`
- Communication: `DashboardPage::moduleClicked` → `MainWindow::navigateTo`

### New Design

**DashboardPage changes:**
1. Enlarge card from 260x160 → 280x200 to fit status + log area
2. Add a status indicator (colored dot + text) below title
3. Replace description label with a small QTextBrowser (log preview, read-only, ~5 lines)
4. Expose `updateCardStatus(name, connected)` and `appendCardLog(name, html)` public methods
5. Store card widgets in a `QMap<QString, QWidget*>` for lookup

**WebSocketPage changes:**
1. Add new signals:
   - `statusChanged(bool connected)` — emitted on connect/disconnect
   - `logAppended(const QString &html)` — emitted each appendLog call
2. Emit these signals from existing `onConnected()`, `onDisconnected()`, `appendLog()`

**MainWindow changes:**
1. In `initPages()`, connect new WebSocketPage signals to DashboardPage methods:
   - `m_websocketPage::statusChanged` → `m_dashboardPage::updateCardStatus("websocket", connected)`
   - `m_websocketPage::logAppended` → `m_dashboardPage::appendCardLog("websocket", html)`

## Files to Modify

| File | Change |
|------|--------|
| `Sources/Headers/DashboardPage.h` | Add `QMap`, `updateCardStatus`, `appendCardLog`, store card widget pointers |
| `Sources/DashboardPage.cpp` | Enlarge card, add status label + log preview QTextBrowser, implement update methods |
| `Sources/Headers/WebSocketPage.h` | Add `statusChanged`, `logAppended` signals |
| `Sources/WebSocketPage.cpp` | Emit new signals in `onConnected`, `onDisconnected`, `appendLog` |
| `Sources/MainWindow.cpp` | Connect new signals in `initPages()` |

## Card Layout (new)

```
┌──────────────────────────────┐
│ ⚡  WebSocket            ●在线│  ← icon + title + status dot
│ ──────────────────────────── │
│ 12:01:03 已连接到服务器       │  ← log preview (QTextBrowser, 5 lines max)
│ 12:01:05 ← Hello            │
│ 12:01:06 → Response         │
│                              │
└──────────────────────────────┘
```

## Status Indicator
- Connected: green dot `●` + "在线" (color: `#34d399`)
- Disconnected: gray dot `●` + "离线" (color: `#64748b`)

## Log Preview
- Read-only QTextBrowser, fixed max height ~80px
- Monospace font, 8pt
- Auto-scroll to bottom on new content
- Show last N lines only (keep memory bounded)

## Verification
1. Build with CMake + MinGW
2. Launch app → Dashboard shows WebSocket card with gray "离线" status
3. Click card → navigate to WebSocket page → connect → return to Dashboard
4. Verify card shows green "在线" + log content
5. Disconnect → verify card reverts to "离线"
