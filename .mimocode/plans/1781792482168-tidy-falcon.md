# UI 细节优化计划

## 变更 1：翻译界面向下箭头图标
**文件**: `Sources/TranslationPage.cpp:143-159`

当前 QComboBox 使用 CSS border triangle 作为下拉箭头（`border-top: 4px solid #b8c7e6`）。箭头太小不明显。调整 CSS 让箭头更大更清晰，改用 `>` 或 `▼` Unicode 字符替代 CSS triangle。

## 变更 2：清除和翻译按钮默认隐藏，输入后显示
**文件**: `Sources/TranslationPage.cpp:186-203, 274-281`

当前 `m_clearBtn` 和 `m_translateBtn` 始终可见。改为：
- 初始化时 `hide()` 两个按钮
- `onInputTextChanged()` 中根据输入是否为空决定显示/隐藏
- 按钮仅在有文字时显示在输入行最右端（当前已是 `actionColumn` 水平布局在输入框右侧，位置正确）

## 变更 3：设置 - "关闭按钮行为" 改为单选按钮（Radio Button）
**文件**: `Sources/SettingsDialog.cpp:87, 152-163, 170-198`

当前 `closeAction` 是 `ConfigField::Choice`，渲染为 QComboBox。改为：
- `createModuleTab()` 中对 Choice 类型改为创建 `QRadioButton` + `QButtonGroup`（互斥选择）
- 每个选项前有选中标记（QRadioButton 自带圆点指示器）
- `onSave()` 中把 QRadioButton 的选中 index 保存到 QSettings
- 需要在头文件中添加 `#include <QRadioButton>` 和 `#include <QButtonGroup>`

## 变更 4：退出弹窗背景与主界面一致 + 字体可读
**文件**: `Sources/MainWindow.cpp:168-186`

当前 `QMessageBox::question()` 使用系统默认样式，深色背景下文字看不清。改为：
- 使用 `QMessageBox msgBox` 自定义构造
- 设置 stylesheet 匹配主界面主题 (`#191a1c` 背景，`#e2e8f0` 文字)
- 按钮也设置对应样式（保存按钮 `#0ea5e9`，取消按钮 `#374151`）
